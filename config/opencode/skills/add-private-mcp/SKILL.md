# Add Private MCP Server

Add or update an MCP server in private config (`~/.config/dotfiles/private-opencode.json`) so secrets and local-only endpoints stay out of git.

## When to Use

- User asks to add/update/remove an MCP server.
- MCP config should be private.
- User wants setup aligned with dotfiles merge flow.

## Required Inputs

Collect these from the user or existing context:

- `name`: MCP server key (for example `GitLab`, `Slack`)
- `type`: `remote` or `stdio`
- `enabled`: boolean (default `true`)
- `remote` server: `url`
- `stdio` server: `command`, optional `args`, optional `env`

## Secret Handling (Strict)

- Never write secrets to repo-tracked files.
- Write secrets only under `~/.config/dotfiles/private-opencode.json`.
- If required secret values are missing (for example `SOMETHING_TOKEN`), stop and ask the user for the exact value.
- Do not use placeholders or dummy values.

## Workflow

1. Read current private config from `~/.config/dotfiles/private-opencode.json` if it exists.
2. Ensure top-level object has an `mcp` object.
3. Upsert `mcp.<name>` with provided server fields:
   - `remote`: `{ "type": "remote", "url": "...", "enabled": true }`
   - `stdio`: `{ "type": "stdio", "command": "...", "args": [...], "env": {...}, "enabled": true }`
4. Preserve all unrelated private keys and existing MCP entries.
5. Run `bash setup.sh` from the dotfiles repo to regenerate merged OpenCode config.
6. Verify:
   - `opencode mcp list`
   - Optional: `opencode mcp debug <name>` for connection/auth details.
7. Report what was configured and any follow-up auth steps (`opencode mcp auth <name>` if OAuth).

## Validation Rules

- `remote` requires `url`.
- `stdio` requires `command`.
- `args`, when present, must be a JSON array of strings.
- `env`, when present, must be a JSON object of string values.

## Examples

### Remote

```json
{
  "mcp": {
    "Whatever": {
      "type": "remote",
      "url": "https://mcp.example.com/retrieval-tools-server",
      "enabled": true
    }
  }
}
```

### Stdio

```json
{
  "mcp": {
    "Example": {
      "type": "stdio",
      "command": "~/example/mcp-server",
      "env": {
        "EXAMPLE_REMOTE_URL": "https://example.com:1234",
        "EXAMPLE_TOKEN": "<user-provided-token>"
      },
      "enabled": true
    }
  }
}
```

## Constraints

- Never edit generated files under `~/.local/share/dotfiles/` directly.
- Never edit `~/.config/opencode/opencode.json` directly unless generated by `setup.sh`.
- Prefer canonical repo files for public config and private files under `~/.config/dotfiles/` for local overrides.
