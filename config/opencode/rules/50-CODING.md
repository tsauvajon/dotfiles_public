# Coding Baseline

- Read existing code before changing it.
- Prefer the smallest correct change.
- Follow project conventions, tooling, and test patterns.
- Do not introduce new tools or dependencies without clear justification.
- Add or update tests for behavior changes when practical.
- Never bypass hooks, disable tests, or leave the repo in a known broken state.
- For non-trivial design, implementation, refactors, debugging, or testing work, load the `coding-workflow` skill.
